# Continuous Evaluation with Online Monitors

> **Preview** — Agent evaluation on Gemini Enterprise Agent Platform is subject to the
> "Pre-GA Offerings Terms" in the General Service Terms of the Service Specific Terms, the
> Generative AI Service Specific Terms, and the "Agentic AI Services" Service Specific Terms.
> Pre-GA products are provided "as is" and might have limited support.

Online monitoring continuously assesses the quality of your agents **in production**. It helps
you identify **quality drift** — an observable decrease in agent performance over time caused by
changes in user behavior or external data. Online Monitors asynchronously score live traces
using both predefined and custom metrics.

Related: [OFFLINE_EVALUATION.md](./OFFLINE_EVALUATION.md) (historical traces) ·
[MANAGE_METRICS.md](./MANAGE_METRICS.md) · [QUALITY_ALERTS.md](./QUALITY_ALERTS.md) ·
[ANALYZE_RESULTS.md](./ANALYZE_RESULTS.md)

## Before you begin

- Deploy your agent (see *Deploy an agent*).
- Ensure **Cloud Trace** is enabled for your project.
- *(Optional)* To create monitors programmatically, initialize the Agent Platform SDK (see
  *Evaluate your agents*).

> **Security disclosure:** Online evaluation relies on the project-level service account
> (P4SA). Any user with permission to create an `OnlineEvaluator` can attach it to **any** agent
> in the same project. To avoid privilege escalation, restrict `OnlineEvaluator` creation to
> authorized administrators.

## Telemetry requirements

Online monitoring requires the same OpenTelemetry signals as offline evaluation.

### Invoke agent span

| Attribute | Meaning |
|---|---|
| `gen_ai.agent.name` | The identifier for the agent. |
| `gen_ai.agent.description` | A brief description of the agent's purpose. |
| `gen_ai.conversation.id` | A unique identifier for the specific conversation session. |

### Inference events

The `gen_ai.client.inference.operation.details` event must capture:

| Attribute | Meaning |
|---|---|
| `gen_ai.input.messages` | The prompts sent to the agent. |
| `gen_ai.output.messages` | The responses generated by the agent. |
| `gen_ai.system_instructions` | The underlying system prompts. |
| `gen_ai.tool.definitions` | Metadata about any tools available to the agent. |

### ADK telemetry environment variables

```bash
OTEL_SEMCONV_STABILITY_OPT_IN='gen_ai_latest_experimental'
OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT='EVENT_ONLY'
```

### Recording media in Cloud Storage

For multimodal data (images, large documents), record inputs/outputs in a bucket instead of
embedding them in trace spans:

```bash
OTEL_INSTRUMENTATION_GENAI_UPLOAD_FORMAT='jsonl'
OTEL_INSTRUMENTATION_GENAI_COMPLETION_HOOK='upload'
OTEL_INSTRUMENTATION_GENAI_UPLOAD_BASE_PATH='gs://STORAGE_BUCKET_NAME/PATH'
```

## How online monitors work

Online Monitors run on a scheduled evaluation loop, **typically every 10 minutes**:

1. **Query** — samples data from Cloud Trace and Cloud Logging based on your filters.
2. **Evaluate** — runs configured metrics using the Agent Platform Evaluation Service.
3. **Report** — writes results back to Cloud Logging and exports numeric scores to Cloud
   Monitoring.

## Create an online monitor

1. In the Google Cloud console, navigate to **Agent Platform > Agents > Evaluation**.
2. Select the **Online monitors** tab and click **New monitor**.
3. **Specify Filter Traces:**
   - **Agent engine** — select the agent to monitor.
   - **Filter criteria** — evaluate **All traces** or apply specific **Filter criteria**.
4. **Define Filter Criteria** (if filtering):
   - **Initial Inspection** — select a timeframe (e.g., *Last 1 day*) to preview matching
     production traces.
   - **Filters** — target specific traffic by properties such as **Duration** (e.g.,
     `Duration > 2`) or **Token usage**.
5. **Configure Metrics** — add metrics to track continuously, such as **Safety**.
6. **Set Sampling:**
   - **Sampling percentage** — what percentage of live traffic to evaluate.
   - **Max samples per run** — a cap to manage evaluation costs.
7. Click **Create**.

## Manage monitors

From the **Online monitors** list:

- **Status toggle** — More options (⋮) → **Enable** / **Disable** to pause evaluation without
  deleting the configuration.
- **Pause and resume** — temporarily stop evaluation via More options (⋮).
- **Duplicate** — create a new monitor pre-filled from an existing one.
- **View traces** — the **View traces** link in the **Sampled traces** column navigates to the
  filtered traces in the agent's Traces tab.

## View results in the observability dashboard

1. Navigate to **Agent Platform > Agents**, then **Deployments** in the left nav.
2. Select your agent.
3. In the **Dashboard** view, select the **Evaluation** subsection to view time-series charts
   for configured metrics (response quality, safety, hallucination rates, etc.).

## View results for individual traces

1. Navigate to the agent's **Traces** tab.
2. Select a session or trace to open the details panel.
3. Select the **Evaluation** tab to view scores and rationales for that interaction.

## Troubleshoot online monitors

If your monitor is active but no results appear:

- **Verify telemetry** — confirm the agent exports the required OpenTelemetry spans and events.
  Check Cloud Trace for live traces containing the `gen_ai.` attributes.
- **Check filters** — review the monitor's filter criteria; use **Initial Inspection** to
  confirm filters match production traffic.
- **Check internal logs** — Online Monitors write diagnostics to Cloud Logging. Find them in
  Logs Explorer:

```text
resource.labels.online_evaluator="projects/YOUR_PROJECT_ID/locations/YOUR_REGION/onlineEvaluators/YOUR_MONITOR_ID"
# Or search by trace or agent
labels.trace="YOUR_TRACE_ID"
labels.reasoning_engine_id="YOUR_AGENT_ID"
```
